Breadth First Search (BFS)

Algoritmo de busqueda sin informacin que se expande y examina todos los nodos

Se comienza desde la raz y se exploran todos los vecinos de ese nodo. A continuacion, para cada uno de esos vecinos se exploran los vecinos adyacentes que no hayan sido visitados hasta terminar de recorrer todo el grafo.

Pasos

  1. Seleccionar un nodo y agregar a la cola (FIFO)
  2. Sacar de la cola el primer elemento y ver los nodos que estan relacionados (vecinos).
  3. Agregar a la cola los nodos relacionados que no fueron visitados
  4. Volver a reptetir desde el paso 2 hasta que no haya mas elementos en la cola

Depth First Search (DFS)

Algoritmo de busqueda sin informacin que se expande y examina todos los nodos

Se comienza desde el nodo raiz y se va expandiendo con todos los nodos que va localizando de forma concurrente y en un camino concreto. Es decir, que en vez de ver todos los nodos relacionados al mismo tiempo solamente mira de a uno y va por ese camino hasta que no haya mas nodos para ver. Luego hace un backtracking regresando al primer nodo para seguir con los otros nodos

Pasos

  1. Seleccionar un nodo como raiz
  2. Marcar el nodo como visitado.
  3. Iterar por todos los nodos hijos inmediatos del nodo, si hay un nodo que no es visitado llamar recursivamente a la funcion de ese nodo hijo antes de mirar el siguiente nodo hijo inmediato

Detector de Ciclos (Cycle Detection) - Grafos Unidireccionales

Un Grafo Ciclo es un grafo que consiste en un camino simple cerrado, en el que no se repite ningun vertice salvo el primero con el ultimo.

Para la deteccion de un grafo ciclico se puede usar BFS o DFS

En DFS lo que se hace es chequear si hay un nodo que ya se visito antes y si ese nodo es el nodo padre. De ser asi, es un grafo cicliclo

Detector de Ciclos (Cycle Detection) - Grafos Direccionales

Se sigue el mismo principio que el de arriba. Solo que s mueve en una direccion en particular.

Se empieza desde un nodo, se recorre y se vuelve al mismo nodo y no hay otro que este haya sin visitar entonces es un grafo ciclico.

Hay que tener en cuenta que dependiendo de las direcciones puede ser o no un grafico ciclico

Grafo Bipartito (Bipartite Graph)

Un grafo bipartito es un grafo cuyos vertices se pueden separar en 2 conjuntos disjuntos (No tienen elementos en comun) distintos de manera que las aristas no pueden relacionar vertices de un mismo conjunto

En un grafo bipartito:

  • Se pueden dividir todos los vertices del grafo en 2 conjuntos
  • Por lo tanto, todas las uniones del grafo van entre el set1 y el set2

Para armarlo lo que se hace es poner un nodo en un conjunto y los del siguiente nivel en otro.

Generalemente se incumple la norma cuando:

  1. Si se recorre nodo por nodo (1 -> 2 -> 3..) y el ultimo nodo conecta con el primero y cae dentro del mismo conjunto, entonces se incumple
  2. Si se van cargando por niveles, pero en algun nivel se conecta entre si entonces incumple la norma

Directed Acyclic Graph (DAG)

Es una ordenacin lineal de todos los nodos de G que satisface que si G contiene la arista dirigida uv entonces el nodo u aparece antes del nodo v.

La condicin que el grafo no contenga ciclos es importante, ya que no se puede obtener ordenacin topolgica de grafos que contengan ciclos.

Usualmente, para clarificar el concepto se suelen identificar los nodos con tareas a realizar en la que hay una precedencia a la hora de ejecutar dichas tareas. La ordenacin topolgica por tanto es una lista en orden lineal en que deben realizarse las tareas.

  1. Lo que se hace primero es empezar por un nodo que no tenga aristas entrantes, y se recorre en profundidad.

Para el ejemplo: 1 -> 4 -> 5 2. Una vez que se llega al ultimo nodo, y ya no hay otro para visitar y se tiene que regresar al nodo padre, se agrega ese nodo a una lista. Se usa push-front, es decir, se van agregando los nodos a la posicion 0 de la lista

  1. Si desde el nodo padre no tiene mas nodos para visitar, se agrega a la lista, caso contrario se visitan en prfundidad cada uno de sus neighbors y se van agregando a la lista siguiedo el item 2

Para el ejemplo:

  3. Se vuelve al Nodo 1, como este tiene nodos a visitar (nodo 2) se visita este, luego el nodo 3.
  4. Como el Nodo 3 no tiene mas nodos que visitar (ya que el 5 fue visitado) se agrega a la lista.
  5. Y asi sucesivamente...

Estructura de datos para conjuntos disjuntos (DSU data Structure)

Teoria

Un conjunto disjunto es una estructura de datos que realiza un seguimiento de un conjunto de elementos divididos en varios subconjuntos disjuntos (que no se superponen).

En otras palabras, un conjunto disjunto es un grupo de conjuntos donde ningn elemento puede estar en ms de un conjunto.

Admite dos tipos de funciones:

  1. Find(element): Determina en que conjunto se encuentra un elemento en particular y devuelve el representante de ese conjunto. Un elemento de ese conjunto actua como "representante" de ese conjunto

  2. Union(s1, s2): fusiona dos subconjuntos diferentes en un solo subconjunto y el representante de un conjunto se convierte en el representante del otro.

Implementacion

Los Disjoint-set son estructuras de datos donde cada conjunto est representado por un rbol de datos en el que cada nodo tiene una referencia a su padre y el representante de cada conjunto es la raz del rbol de ese conjunto.

Por ejemplo, si tenemos los conjuntos: S1, S2, S3, S4, S5

Cada conjunto contiene inicialmente solo un elemento cada uno por lo que su puntero apunta asi mismo:

Es decir, tenemos una lista de relaciones:

Si hacemos un find(i) = i ya que i es el representante de Si

si hacemos un union(S3, S4) S3 y S4 se fusionar en un conjuto inconexo S3. Donde ahora, el representante de S3 sera el representante de todo el conjunto.

Es decir:

  • S1 = {1}
  • S2 = {2}
  • S3 = {3, 4}
  • S5 = {5}

Si hacemos un find(4) = 3 ya que 3 se convierte en el representante del conjunto resultante. Las relaciones quedan:

si hacemos union(S1, S2)

Quedaria: S1 = {1, 2}, S3 = {3, 4} and S5 = {5}

y si hacemos find: find(2) = find(1) = 1

si hacemos union(S3, S1)

Quedaria: S3 = {1, 2, 3, 4} and S5 = {5}

y si hacemos find: find(1) = find(3) = 3

Estas operaciones asi como estan son muy sencillas pero no es la mejor manera ya que pueden hacer que el arbol resultante pueda ser muy desvalanceado.

Para mejorar esto, se puede mejorar con las optimizaciones de Union por rango (Union by rank) o Compresion del camino (Path compression Optimisation)

Union Por Rango

Teoria

Consiste en siempre aadir el rbol ms pequeo a la raz del rbol ms grande.

Como la profundidad del rbol afecta el tiempo de ejecucin del algoritmo, el rbol con menor profundidad es aadido a la raz del rbol con mayor profundidad, el cual aumenta su profundidad solo si sus profundidades son iguales

Los rboles con un solo elemento tienen rank igual a cero, y cualquiera dos rboles del mismo rank r son combinados, el rank resultante es r+1.

El trmino rank se usa en vez de profundidad porque este deja de ser igual a la profundidad si se usa la compresin de camino

Ejemplo

Partiendo de nuestra lista de ranks:

  • Todos los nodos que no son representantes tienen rank=0
  • Todos los nodos que son representantes tienen rank=len()

  1. Al hacer un union rank de los 2 arboles, el arbol de rank mas pequeo se uno con el arbol de rank mas grande.

  2. Una vez que se unen, se vuelve a actualizar la tabla de ranks

Ejemplo de cdigo

 function CrearConjunto(x)
     x.padre := x
     x.rank  := 0

 function Unin(x, y)
     xRaz := Buscar(x)
     yRaz := Buscar(y)
     if xRaz == yRaz
        return
     //Ya que no estn en el mismo conjunto, se unen.

Compresion del camino

Es una forma de aplanar la estructura del rbol cuando se llame la funcion find

La idea es que cada nodo visitado en el camino hacia la raz puede ser aadido directamente a la raz ya que todos ellos comparten el mismo representativo.

Para lograr este efecto, mientras se llama la funcin find recursivamente mientras se mueve hacia la raz, este cambia la referencia del padre del nodo a la raz que encuentre.

El rbol resultante es ms aplanado, acelerando operaciones futuras no solo en estos elementos sino tambin en aquellos que referencian a estos.

Minimum Spanning Tree (Arbol de expansin minimo)

Un MST de un grafo con pesos (ponderado), es un subgrafo el cual contiene todos los vertices del grafo inicial, los cuales estan unidos por aristas sin formar ciclos.

Cada arista tiene asignado un peso proporcional y se usa para asignar un peso total al arbol de expansion computanto la suma de los pesos de todas las aristas.

Un Arbol de expansion minimo es el que pesa menos o igual que todos los otros arboles de expansin.

Un grafo puede tener multiples Arboles de Expansin

El MST es til en la planificacin de redes de comunicacin eficientes, diseo de circuitos elctricos y en cualquier situacin en la que se quiera encontrar una manera eficiente y de bajo costo para conectar un conjunto de puntos o vrtices.

Existen varios algoritmos para encontrar el MST de un grafo ponderado, entre ellos:

  • Algoritmo de Prim: Comienza desde un vrtice inicial y en cada paso agrega la arista de menor peso que conecta un vrtice ya en el MST con un vrtice fuera del MST.

  • Algoritmo de Kruskal: Ordena todas las aristas por peso y las va agregando al MST de manera creciente, asegurndose de que no se formen ciclos.

Algoritmo de Prim

El algoritmo encuentra un subconjunto de aristas que forman un rbol con todos los vrtices, donde el peso total de todas las aristas en el rbol es el mnimo posible.

  • Es un algoritmo codicioso / avaro
  • Usa una cola de prioridad

Teoria

Propiedades del subgrafo:

  • Todos los vertices del grafo original existen en el subgrafo
  • Todos los nodos estan conectados en el subgrafo
  • No hay ciclos en el subgrafo

Pasos

  1. Se comienza seleccionando un nodo arbitrario como vertice inicial

  2. Se agrega el nodo seleccionado a una lista MST_set=[] que es la que indica que nodos ya se visitaron

  3. Me fijo que vertices estan conectados al vertice actual y los agergo en una lista de priodidad como (weight, next_vertex)

  4. De la lista, saco el valor que tenga el weight mas chico y que next_vertex no este en MST_set (NOTA: Quizas tengo que corroborar que al seleccionar este nodo no haga un ciclo)

  5. Acumulo el weight del valor que saque (Que va a ser el minimum spanning tree value)

  6. Visito el nodo de la tupla que saque, y vuelvo a repetir desde el paso 2.

Algoritmo de Kruskal

Teoria

Es un algoritmo para encontrar un arbol recubridor minimo en un grafo ponderado.

Caracteristicas:

  • Algoritmo avaricioso
  • Guarda el grafo como una lista de aristas
  • Usa DSU para detecin de Ciclos

Partiendo del grafo:

Pasos:

  1. Se parte del grafo sin aristas

  1. Guardar en una lista la tupla (weight, from_node, to_node)

  2. Ordenar la tupla de menor a mayor segun el peso (weight)

  1. Se popea el element con peso menor de la lista. Si esta arista no forma ningun ciclo en el grafo, entonces se agrega y se suma el peso

b. (2, 1, 3): Como no forma ningun ciclo, se une nodo_1 con nodo_3 y se suma el peso 2

d. (2, 2, 3): Como forma un ciclo, se saltea y se pasa al siguiente elemento

  1. El resultado del MST sera la suma total de los pesos en cada proceso que se agrega la arista o edge

Es un algoritmo para encontrar el camino mnimo en grafos dirigidos ponderados.

El algoritmo encuentra el camino entre todos los pares de vrtices en una nica ejecucin.

El algoritmo de Floyd-Warshall es un ejemplo de programacin dinmica.

Introduccin

El algoritmo de Warshall es un ejemplo de algoritmo booleano.

A partir de una tabla inicial compuesta de 0's (no hay correspondencia inicial en el grafo) y 1's (hay una correspondencia, llamase flecha, entre nodos), obtiene una nueva matriz denominada Matriz de Clausura Transitiva en la que se muestran todas las posibles uniones entre nodos, directa o indirectamente.

Es decir, si de A a B no hay una flecha, es posible que si haya de A a C y luego de C a B. Luego, este resultado se ver volcado en la matriz final.

Algoritmo

Supongamos que tenemos el siguiente grafo:

  1. Lo primero que se hace es armar una matriz con los pesos de cada uno (matriz de distancias) y otra matriz con los nodos (matriz de recorridos)

En la primer matriz indica los pesos que hay para ir de un nodo al otro, y en la matriz de distancias se comienza poniendo el nodo que haya en la columna. D0 indica las relaciones directas de nodo a nodo

Tanto la primer fila como la primer columna indican los pesos que hay para ir de forma directa hasta el otro nodo

Problema del viajero (TSP)

El problema del viajero responde a la siguiente pregunta: dada una lista de ciudades y las distancias entre cada par de ellas, cul es la ruta ms corta posible que visita cada ciudad exactamente una vez y al finalizar regresa a la ciudad origen?. Esto tambien se lo conoce como ciclo Hamiltoniano

Enfoque usando Brute Force

Lo que se plantea con fuerza bruta es comparar todas las posibles permutaciones de las rutas o caminos para derterminar la solucion mas optima (camino mas corto)

Ejemplo:

* A -> B -> C -> D -> A
* B -> C -> D -> A -> B
* A -> C -> D -> B -> A

Este es un problema NP-hard, por lo que tiene una demanda un tiempo de O(n!)

Arbol con distintas posibilidades:

Lo que vamos a hacer es ir visitando cada ciudadad, y una vez que hayamos visitado todas las ciudades se va retornando el costo de ir desde la ultima ciudad hasta el source node.

Para el ejemplo:

A -> B -> C -> D -> A

Una vez que llegamos a D (visitamos todas las ciduades) devolvemos el costo de pasar de D a A. En este caso, 25.

y mientras vamos subiendo, lo que hacemos es ir sumando los costos de pasar del nodo anterior al nodo actual.

En caso de que nos encontremos en un punto donde se unos 2 nodos. Como sucede en B se toma la rama que tenga el minimo peso. en este caso es C, por lo que para pasar de C -> B es de 30

En el ejemplo, solo tomamos B para pasar a A, pero habria que tambien comprar los demas nodos.

Enfoque usando Programacion Dinamica (DP)

Al usar programacion dinamica, se mejora el tiempo pasando de O(n!) a O(n22n)

A primera vista, esto puede no parecer una mejora sustancial, sin embargo, ahora hace factible resolver este problema en grficos con hasta aproximadamente 23 nodos en una computadora tpica.

Que es la programacion dinamica?

La programacin dinmica es un mtodo para reducir el tiempo de ejecucin de un algoritmo.

Por ejemplo, el camino ms corto entre dos vrtices de un grafo se puede encontrar calculando primero el camino ms corto al objetivo desde todos los vrtices adyacentes al de partida, y despus usando estas soluciones para elegir el mejor camino de todos ellos.

Programacion Recursiva vs Dinamica

Por ejemplo, para calcular fibbonachi, si lo hacemos de forma recursiva, vamos a calcular el fibbonaci de 5, luego a su vez el de 4 y 3 y asi lo iriamos repitiendo. Lo cual, estariamos volviendo a calcular el fibbonaci de numeros que ya calculamos en algun otro momento. Estoy tiene un coste computacional de O(2n).

Una de las formas de solucionar este problema es con la programacin dinamica, que lo hace hace es guardar los valores calculados para luego volverlos a usar sin necesidad de calcularlos. Similar a una cache

La programacion dinamica tiene 2 items importantes:

  1. Memoizacion: Una vez que calculamos algun valor, lo guardamos en una matriz o array para utilizarlo en un posterior proceso. Este problema baja el coste operacional pero puede darnos un maximun call stack cuando se tiene que calcular numeros altos

  2. Linealidad en vez de llamar funciones recursivas, se trata de empezar a calcular los valores mas de bases y con un "for" completar los siguintes. Por ejemplo, para calcular el fibonacci de 5, podemos guardar en una lista el fibonaci de 0 y 1, e ir calculando el fibbonaci de 2 a 5 sin necesidad de usar recursividad.

Este enfoque simplemente tiene como proposito guardar en una matriz, el bitmask, la ciudady la distancia. Asi, se evita volver a calcular la distancias en caso de haberlas calculado con anterioridad.

Esto puede resultar confuso, pero recordemos que las distancias siempre la calculamos de atras para adelantes.

Supongamos que tenemos estas rutas:

* A -> B -> C -> D -> A
* A -> C -> B -> D -> A

Antes de llegar al nodo D, en ambos casos vamos a tener: 0111, y en ambos casos al querer calcular la distancia de D, seria lo mismo, ya que desde D obtengo la distancia a A y eso lo voy pasando a los nodos anteriores.

Ejemplo, cuando hago el primer camino: A -> B -> C, puedo guardar: dp[0111][D] = dist_D

cuando llamo al segundo ejemplo, no hace falta volver a calcular la distancia de D, ya que va a ser la misma que calculamos anteriormente ya que no depende de los nodos anteriores sino de los nodos siguientes.

Algoritmo Flood Fill

El algoritmo flood fill - algorimo de relleno - determina el area formada por elementos continuos en una matriz multidimencional. Funciona como se se hechara agua, va rellenando los huecos vacios hasta completar toda el area.

Es decir, partimos de un nodo inicial y nos vamos moviendo en todas las direcciones hasta llenar todos los huecos.

int n, m;

// Definimos los tipos de movimientos
int dx[4] = {0, 0, 1, -1};
int dy[4] = {1, -1, 0, 0};

void flood_fill(int x, int y) {
	a[x][y] = 0;
	for (int i = 0; i < 4; i++) {
		int xx = x + dx[i];
		int yy = y + dy[i];

		if (xx >= 0 && xx < n && yy >= 0 && yy < m && a[xx][yy] == 1) {
			flood_fill(xx, yy);
		}
	}
}

const int N = 100;

int a[N][N], vis[N][N];
int col_cnt[N];

int n, m;

int dx[4] = {0, 0, 1, -1};
int dy[4] = {1, -1, 0, 0};

void flood_fill(int x, int y, int col) {
	a[x][y] = col;
	col_cnt[col]++;
	vis[x][y] = 1;
	for (int i = 0; i < 4; i++) {
		int xx = x + dx[i];
		int yy = y + dy[i];
		if (xx >= 0 && xx < n && yy >= 0 && yy < m && vis[xx][yy] == 0 && a[xx][yy] == 1) {
			flood_fill(xx, yy, col);
		}
	}
}

int main(){
	freopen("input.txt", "r", stdin);
	freopen("output.txt", "w", stdout);

	cin >> n >> m;
	for (int i = 0; i < n; i++) {
		for (int j = 0; j < m; j++) {
			cin >> a[i][j];
		}
	}

	int total_count = 0;

	for (int i = 0; i < n; i++) {
		for (int j = 0; j < m; j++) {
			if (a[i][j] == 1 && vis[i][j] == 0) {
				total_count++;
				flood_fill(i, j, total_count);
			}
		}
	}

	int largest_island = 0;

	for (int i = 1; i <= total_count; i++) {
		largest_island = max(largest_island, col_cnt[i]);
	}

	for (int i = 0; i < n; i++) {
		for (int j = 0; j < m; j++) {
			if (a[i][j] == 0) {
				// can be converted
				set<int> St;
				for (int k = 0; k < 4; k++) {
					int ii = i + dx[k];
					int jj = j + dy[k];
					if (ii >= 0 && ii < n && jj >= 0 && jj < m) {
						St.insert(a[ii][jj]);
					}
				}
				int ans = 1;
				for (auto x : St) {
					ans += col_cnt[x];
				}
				largest_island = max(largest_island, ans);
			}
		}
	}
	cout << largest_island;
	return 0;
}

Backages con DFS tree

Breve introduccin:

En el diagrama a continuacin, si se aplica DFS en este grfico, se obtiene un rbol que se conecta mediante bordes verdes dando la union de los nodos 1 2 4 6 3 5 7 8.

Conceptos introductorios

  • Tree Edge: (Arista de rbol) Es una arista que est presente en el rbol obtenido despus de aplicar DFS en el grafo. Todas las aristas verdes son aristas de rbol.

  • Forward Edge (Arista Avanzada): Es una arista (u, v) tal que v es un descendiente pero no forma parte del rbol DFS. Una arista de 1 a 8 es una arista forward edge.

  • Back Edge (Arista de Retroceso): Es una arista (u, v) tal que v es el ancestro del nodo u pero no forma parte del rbol DFS. La arista de 6 a 2 es una arista de backedge. La presencia de una arista de retroceso indica un ciclo en el grafo dirigido.

  • Cross Edge (Arista Cruzada): Es una arista que conecta dos nodos de manera que no tienen relacin de ancestro y descendiente entre ellos. La arista del nodo 5 al 4 es una cross edge.

Se tiene el siguiente arbol el cual se aplica DFS.

al recorrer el arbol hace lo siguiente: 1 -> 2 -> 3 -> 4 -> 5 -> 6

Como se puede ver las aristas que conectan 1 con 3 o 4 con 6 se llaman backedges ya que no se usan para volver al nodo original. Esto ademas sirve para detetar la cantidad de ciclos que pueda llegar a haber en un grafo. En este casos hay 2 ciclos.

Algoritmo para detectar backedges:

Supongamos que partimos del nodo 4 el primer paso para detectar un backedge podria ser poner en una pila nodos los nodos padres.

Al hacer DFS, si un nodo esta visitado y se encunetra dentro de los nodos padres y ademas ese nodo NO es un padre directo significa entonces que hay un backedge.

La cantidad de backedges puede calcularse con la formula:

m(n1)

  • m: Cantidad de vertices
  • n: Cantidad de nodos

Codigo

vector<int> gr[N];
int vis[N];
bool cycle = false;

void dfs(int cur, int par) {
  // Lo marcamos como visitado y lo cargamos en el call stack
  // Que sera los nodos padres
	vis[cur] = 1;
	for (auto x : gr[cur]) {
		if (vis[x] == 0) {
			dfs(x, cur);
		}
		else if (x != par && vis[x] == 1) {
			// backedge
			cycle = true;
		}
	}
  // Lo dejamos como visitado pero con 2 indicamos que ya no
  // corresponde al call stack
	vis[cur] = 2;
	return;
}

int main()
{
	freopen("input.txt", "r", stdin);
	freopen("output.txt", "w", stdout);
	int n, m;
	cin >> n >> m;
	for (int i = 0; i < m; i++) {
		int x, y;
		cin >> x >> y;
		gr[x].push_back(y);
		gr[y].push_back(x);
	}

	for (int i = 1; i <= n; i++) {
		if (vis[i] == 0) {
			dfs(i, 0);
		}
	}
	if (cycle) {
		cout << "Yes cycle found";
	}
	else {
		cout << "Not found";
	}
	return 0;
}

Bridges and Articulation Points

Articulation Points:

En un grafo, se considera un Articulation Point a un vertice que al eliminarlo da como resultado un aumento de componentes en el grafo.

Ejemplo:

  • Al eliminar el nodo 3, nos quedan 2 componentes en el grafo.

  • Si eliminamos el Nodo 4 quedan 2 componentes en el grafo.

Una de las maneras de encontrar el Articulation Point es usar fuerza bruta para determinar si un vertice es un punto de articulacion o no. Lo que se hace es ir eliminando cada vertice y luego contar el nmero de componentes conectados en el grafo. Si el nmero aumenta, entonces es un punto de articulacion.

For every vertex V in the graph G do
  Remove V from G
   if the number of connected components
   increases then V is an articulation point
  Add V back to G

Es el minimo discovered time de un nodo al que podemos atravesar usando como mximo un backedge en el subtree actual.

Ejemplo: Supongamos que tenemos este grafo, donde el discovery time es el numero que aparece en rojo (que corresponde para el ejemplo al N del nodo)

El subtree se le dice al nodo actual y a los nodos que tenemos por debajo. Por ejemplo, el Subtree del nodo 2, es el Nodo 2 y Nodo 3

El lowest discovered time va a ser el tiempo en el que desde un nodo de un subtree puedo llegar a un ancestro (Nodo que ya fue descubierto en el DFS) mediante un backedge.

Como podemos ver, en Nodo 3 tiene como ancestro al Nodo 1 por lo que por un backedge puedo llegar a este. Como esto es asi, el lowest time del Nodo 3 va a ser el discovered time del nodo 1

dist[ancestor]<dist[current.node]

Para el Nodo 2, su Subtree es el Nodo 2 y Nodo 3, como en el subtree hay un solo backedge el lowest time, sera el discovered time del nodo al que se puede acceder por ese backedge. si hubiera varios, entonces en ese caso se eligiria el menor.

Entonces, el lowest time sera el discovered time del Nodo 1

Para el Nodo 1, Como en el subtree el backedge apunta a si mismo, entonces el menor discovered time va a ser su propio discovered time. En este caso es 1

Regla para el nodo Root

Si el numero de hijos de un nodo root es > 1 en un arbol usando DFS entonces el nodo root es un articulation point

Ejemplo:

Usando DFS vemos que el root node solo tiene un hijo, ya que el nodo 3 esta conectado al nodo 1 por medio de un backedge entonces No se cumple la condicion por lo que el nodo root NO es un articulation point

Codigo:

const int N = 1e5;

vector<int> gr[N];

int vis[N], disc[N], low[N], tme = 1;

vector<pair<int, int>> bridges;
set<int> arti_points;

void dfs(int cur, int par) {
	vis[cur] = 1;
	disc[cur] = low[cur] = tme++;
	int child = 0;
	for (auto x : gr[cur]) {
		if (!vis[x]) {
			dfs(x, cur);
			child++;
			// we know low and disc of x
			low[cur] = min(low[cur], low[x]);
			// bridges
			if (low[x] > disc[cur]) {
				bridges.push_back({cur, x});
			}
			// articulation points
			if (par != 0 && low[x] >= disc[cur]) {
				arti_points.insert(cur);
			}
		}
		else if (x != par) {
			// backedge
			low[cur] = min(low[cur], disc[x]);
		}
	}
	// root is an arti or not
	if (par == 0 && child > 1) {
		arti_points.insert(cur);
	}
	return;
}


int main()
{
	freopen("input.txt", "r", stdin);
	freopen("output.txt", "w", stdout);
	int n, m;
	cin >> n >> m;
	for (int i = 0; i < m; i++) {
		int x, y;
		cin >> x >> y;
		gr[x].push_back(y);
		gr[y].push_back(x);
	}
	dfs(1, 0);
	for (auto x : arti_points) cout << x << '\n';
	for (auto x : bridges) {
		cout << x.first << " " << x.second << '\n';
	}
	return 0;
}

Orden Topologico y Componentes fuertemente conectados

Clasificacin Topolgica (topological Sorting)

La clasificacin topolgica para el grfico acclico dirigido (DAG) es una ordenacin lineal de vrtices tal que para cada borde dirigido u v, el vrtice u viene antes de v en el orden.

Es decir, ordenamos los nodos de tal forma que siempre se cumpla la permutacion.

Por ejemplo:

Una forma de ordenamiento podria ser: 1 2 3

Ya que se cumple: 1 -> 2 -> 3

Si lo ponemos: 2 1 3 solo se cumple apra 2 -> 3pero no se estaria cumpliendo para 2 -> 1 ya que primero se parte de 1 para llegar a 2.

Se usa mucho para encontrar dependencias entre diferentes Tareas.

Otro ejemplo:

Se podria ordenar como: D A E B C F o D B F A C E, etc.

Cuando un grafo tiene ciclos, se puede producir un Deadlock ya que no tendria solucion porque todos dependen de todos.

La clasificacion topologica podria:

  • No ser unica
  • Podria no existir si el grafo contiene ciclos, ya que habia una dependencia circular
  • Hay diferencias entre DFS y la clasificacion topologica

Algoritmo

Para hacer el Algoritmo, lo que se hace es algo similar a DFS.

Ser recorren los nodos y sus hijos y una vez que se llega a un nodo donde no hay nada mas para recorrer se pushea a una lista y a medida que va yendo para atras se van pusheando los demas nodos que no tienen otro camino.

Ejmeplo:

  1. Comienzo del nodo 1
  2. Voy hacia el nodo 2
  3. Voy al nodo 4, como no tengo otro camino lo puseo a una lista
  4. Regreso al 2, y sigo con el nodo 5
  5. Como no hay otro camino puseo el 5 a la lista y regreso al nodo 1
  6. Como no hay mas nodos, pusheo los nodos que faltan recorrer

Quedando: 4 5 2 1 3

Una vez teniendo la lista, lo que se hace es cambiarle el orden: 3 1 2 5 4

Genralmente, como paso en el nodo 3, se recorren los nodos no visto y se hace un DFS sobre esos nodos y automaticamente se va pusheando a la lista los nodos faltantes:

Codigo

vector<int> gr[N];
vector<int> order;
int vis[N];

void dfs(int cur, int par) {
	vis[cur] = 1;
	for (auto x : gr[cur]) {
		if (!vis[x]) {
			dfs(x, cur);
		}
	}
	// I am here because I came back from the subtree
	order.push_back(cur);
	return;
}

int main()
{
	freopen("input.txt", "r", stdin);
	freopen("output.txt", "w", stdout);

	int n, m;
	cin >> n >> m;
	for (int i = 0; i < m; i++) {
		int x, y;
		cin >> x >> y;
		gr[x].push_back(y);
	}

   // Recorro TODOS los nodos para asegurarme de que se llame al dfs
   // y se rrecorra todo el grafo apra evitar que hayan nodos sin ver
	for (int i = 1; i <= n; i++) {
		if (!vis[i]) dfs(i, 0);
	}
	reverse(order.begin(), order.end());
	return 0;
}

Topological order

Para generar un ordenamiento, lo que se hace es lo siguiente:

  1. Generar el grafo Condensado

  2. Hacer un reverse del grafo condensado

  3. Tomar un sink node del grafo condensado revertido (que seria un source del condensado original)

  4. Visitar todos los nodos con DFS relacionado a al nodo sink del grafo condensado revertido seleccionado.

  5. Marcar ese nodo sink como visitado y tomar otro

  6. Repetir todo el proceso hasta que no haya mas nodos por visitar

const int N = 1e5;

vector<int> gr[N], grr[N];
int vis[N], col[N];
vector<int> order;

vector<int> components[N];

// topological sorting
void dfs1(int cur) {
	vis[cur] = 1;
	for (auto x : gr[cur]) {
		if (!vis[x]) dfs1(x);
	}
	order.push_back(cur);
}

void dfs2(int cur, int comp) {
	vis[cur] = 1;
	col[cur] = comp;
	components[comp].push_back(cur);
	for (auto x : grr[cur]) {
		if (!vis[x]) dfs2(x, comp);
	}
}

int main()
{
	freopen("input.txt", "r", stdin);
	freopen("output.txt", "w", stdout);
	int n, m;
	cin >> n >> m;
	for (int i = 0; i < m; i++) {
		int x, y;
		cin >> x >> y;
		gr[x].push_back(y);
		grr[y].push_back(x);
	}
	for (int i = 1; i <= n; i++) {
		if (!vis[i]) dfs1(i);
	}
	reverse(order.begin(), order.end());
	memset(vis, 0, sizeof(vis));
	int comp = 1;
	for (auto x : order) {
		// cout << x << " ";
		if (!vis[x]) dfs2(x, comp++);
	}
	for (int i = 1; i <= n; i++) {
		cout << i << " " << col[i] << '\n';
	}
	cout << "Total strongly components are -> " << comp - 1 << '\n';
	// complexity O(n+m)
	return 0;
}

Los arboles son un subconjunto de grafos. Todos los Arboles son grafos, pero un grafo podria no ser un grafo.

Para que un grafo sea considerado un Arbol debe cumplir:

  1. NO puede tener ciclos
  1. No puede tener nodos desconectados

Definicion de arboles

Los arboles son grafos coenctados en los cuales no hay ciclos.

Ademas, cada par de nodos tiene solamente una arista entre ellos.

Por lo que: m=(n1)

Donde:

  • n = Numero de Nodos

  • m numero de aristas

Leaft, child, root, subtree, ancestor LCA

  • ROOT NODE: Es el nodo raiz el cual se desprende todo el arbol. Un arbol solamente puede contener un root node. Puede ser cualquiera de los nodos pero solamente puede haber uno y va a depender del tipo de problema

  • ANCESTORS: Son todos los nodos padres para llegar al ndoo actual partiendo del root node

  • SUBTREE: Es el nodo actual mas todos los nodos hijos hasta llegar a los nodos leaf o los nodos que ya no tienen mas nodos hijos. El subtree siempre depende de que nodo tome como root

  • PARENT: Es el nodo consecutivo anterior para llegar al nodo actual. Depende siempre de que nodo tome como root node

  • LEAFT: Son los nodos los cuales el tamao del subtree es igual a uno. Es decir, que no hay ningun nodo por debajo de ellos.
